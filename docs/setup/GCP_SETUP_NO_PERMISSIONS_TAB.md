# 「権限」タブがない場合の設定方法

## 確認事項

現在、サービス詳細ページに「権限」タブが表示されていない場合は、以下の方法を試してください。

---

## 方法1: 「セキュリティ」タブを確認

1. サービス詳細ページで「**セキュリティ**」タブをクリック
2. 「**プリンシパルとアクセス**」または「**アクセス制御**」というセクションを探す
3. 「**プリンシパルを追加**」ボタンをクリック
4. 以下の設定：
   - **新しいプリンシパル**: 「**allUsers**」と入力
   - **ロールを選択**: 「**Cloud Run 起動元**」を選択
5. 「**保存**」をクリック

---

## 方法2: サービス一覧ページから設定

1. 左側のメニューで「**Cloud Run**」→「**サービス**」をクリック
   - または、ページ上部のパンくずリストで「サービス」をクリック

2. サービス一覧が表示されます
3. 「**gym-reserve-app**」の行を探す
4. 右端の「**三点メニュー（⋮）**」をクリック
   - または、サービス名の右側にあるアイコンを探す

5. メニューから「**アクセス権限の管理**」または「**Manage access**」を選択

6. 「**プリンシパルを追加**」ボタンをクリック

7. 以下の設定：
   - **新しいプリンシパル**: 「**allUsers**」と入力
   - **ロールを選択**: 「**Cloud Run 起動元**」を選択

8. 「**保存**」をクリック

---

## 方法3: IAMと管理から設定（確実な方法）

1. 左側のメニュー（☰）をクリック

2. 「**IAMと管理**」をクリック

3. 「**IAM**」をクリック

4. ページ上部の「**+ プリンシパルを追加**」ボタンをクリック

5. 以下のように設定：
   - **新しいプリンシパル**: 「**allUsers**」と入力
   - **ロールを選択**: ドロップダウンを開いて「**Cloud Run 起動元**」を選択
     - 検索ボックスに「invoker」と入力すると見つけやすいです

6. 「**保存**」をクリック

**注意**: この方法では、プロジェクト全体に権限が付与されますが、Cloud Runサービスには問題なく動作します。

---

## 方法4: URLを直接開く

以下のURLに直接アクセスして、IAMポリシー設定ページを開く：

```
https://console.cloud.google.com/iam-admin/iam?project=gym-reserve-app
```

その後、「**+ プリンシパルを追加**」から設定を行います。

---

## 推奨順序

1. まず「方法1: セキュリティタブ」を試す
2. それでも見つからない場合は「方法3: IAMと管理から設定」を試す（最も確実）
3. それでもダメな場合は「方法2: サービス一覧から設定」を試す

---

## 設定後の確認

設定後、数秒から1分ほど待ってから、以下のURLにアクセス：

```
https://gym-reserve-app-1094873497912.asia-northeast1.run.app/
```

正常にページが表示されれば成功です！
