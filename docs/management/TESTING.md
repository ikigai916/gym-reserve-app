# テスト仕様書 (TESTING)

## 1. テスト方針
現在は Phase 1 (MVP) のため、**手動による機能確認（スモークテスト）** を中心に行います。Phase 2 以降で `pytest` による自動テストを導入予定です。

## 2. 手動テスト手順 (スモークテスト)

### 2.1 サーバー起動確認
1. `uvicorn` または `docker` でサーバーを起動。
2. ブラウザで `http://localhost:8000/health` にアクセスし、`{"status": "ok"}` が返ることを確認。

### 2.2 フロントエンド表示確認
1. `http://localhost:8000/` にアクセス。
2. 画面が正しく表示され、エラーログがブラウザのコンソールに出ていないことを確認。

### 2.3 予約フローの確認
1. トップページで日付を選択。
2. 空いている時間枠の「予約」ボタンをクリック。
3. 必要事項（名前等）を入力して送信。
4. 予約完了メッセージが表示され、一覧に反映されることを確認。

## 3. 異常系チェックリスト
- [ ] 過去の日付を選択して予約を試みる -> エラーになること。
- [ ] 既に予約されている枠を予約しようとする -> エラーになること。
- [ ] 不正なメールアドレス形式を入力する -> バリデーションエラーが出ること。
