# Issue整理方針

## 既存Issueの扱い

### 新しい要件でも必要（維持）
- **Issue #3**: 予約データモデルの拡張（userId, timeSlot, statusの追加）
  - 新しい要件でも必要だが、プランID、メニューID、トレーナーIDなども追加が必要
- **Issue #4**: 時間枠機能の実装（APIとUI）
  - メニューに応じた時間枠の動的表示も必要
- **Issue #5**: 予約キャンセル機能の実装（DELETE API）
  - 新しい要件でも必要
- **Issue #6**: ユーザー情報更新APIの実装（PUT /api/users/{user_id}）
  - 新しい要件でも必要
- **Issue #7**: マイページ機能の実装（UIとAPI連携）
  - 新しい要件では、プラン状況、カルテ履歴なども追加が必要
- **Issue #8**: 予約一覧のフィルタリング機能（userIdによるフィルタ）
  - 新しい要件では、トレーナー向けのフィルタリングも必要
- **Issue #9**: 予約バリデーションの強化（過去日付チェック、重複チェック）
  - 新しい要件では、プラン残り回数のチェックも必要

### 新しい要件に基づいた新規Issue（作成が必要）

1. **ロール管理機能の実装**
   - トレーナー/トレーニーのロール分離
   - ロール別の認証・認可

2. **プラン機能の実装**
   - プランデータモデル
   - プラン購入・登録
   - プラン有効期限管理
   - 残りセッション数管理

3. **メニュー機能の実装**
   - メニューデータモデル（大カテゴリ）
   - メニュー選択機能
   - メニュー管理機能

4. **予約機能の拡張（プラン・メニュー対応）**
   - 予約作成時のプラン選択
   - 予約作成時のメニュー選択
   - プラン残り回数のチェック

5. **カルテ機能の実装**
   - カルテデータモデル
   - カルテ記録機能（トレーナー）
   - カルテ閲覧機能（トレーニー・トレーナー）

6. **管理機能の実装**
   - 予約管理画面
   - 顧客管理画面
   - プラン管理画面
   - メニュー管理画面
